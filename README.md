# Тестовое задание: Сегментация и классификация цифры «4»

Данный репозиторий содержит решение технического задания по автоматизированному поиску и идентификации объектов (цифры «4») на изображениях.

## Содержание задания

В ходе работы были выполнены следующие этапы:
1. **Загрузка изображений**.
2. **Разделение данных** на тренировочную (70%) и тестовую (30%) выборки.
3. **Сегментация изображений** на тренировочной выборке.
4. **Разметка целевого класса**: выделение только цифры «4».
5. **Обучение модели** (CNN) для нахождения цифры «4».
6. **Тестирование** на отложенной выборке.
7. **Расчет метрик**: Acc, Precision, Recall, F1-score и ROC-AUC.
8. **Визуализация**: построение графиков Precision/Recall и ROC-кривой с интерпретацией.
9. **Анализ результатов**: оценка метрик и выбор оптимальной.

## Описание решения

### 1. Подготовка данных и сегментация
Для реализации использован датасет MNIST. Согласно пункту 2 задания, выборка разделена в пропорции 70/30. На этапе сегментации применен метод бинарной маски (бинаризация), где класс «4» помечается как единица (True), а все остальные символы — как ноль (False).

### 2. Архитектура модели
Реализована сверточная нейронная сеть (SimpleCNN), состоящая из слоев свертки (Conv2d), подвыборки (MaxPool2d) и полносвязных слоев (Linear). Финальный слой использует активацию Sigmoid для определения вероятности принадлежности к классу «4».

### 3. Метрики и графики
По завершении обучения рассчитываются ключевые показатели:
* **Accuracy**: общая точность классификации.
* **Precision/Recall**: точность и полнота нахождения объектов «4».
* **F1-score**: сбалансированная оценка между точностью и полнотой.
* **ROC-AUC**: площадь под кривой ошибок.

Согласно пункту 8, в коде генерируются графики ROC и PR-кривых, позволяющие оценить качество модели при различных порогах принятия решения.

## Анализ и выводы (Пункт 9)
* **Недостатки Accuracy**: при дисбалансе классов (четверок всего ~10% от выборки) высокая точность может быть достигнута простым игнорированием целевого класса.
* **Оптимальная метрика**: **F1-score**. Она выбрана как основная, так как эффективно балансирует между риском пропустить цифру «4» (Recall) и риском ложного срабатывания (Precision).
